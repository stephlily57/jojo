# NAYA DASHBOARD — INTERFACE SOUVERAINE

## 1. RÔLE DU DASHBOARD

Le Naya Dashboard est une **interface souveraine**.
Il n’est **pas** un système de contrôle de Naya.
Il n’est **pas** un moteur de décision.
Il n’est **pas** un point de blocage.

Son rôle est :
- observer
- échanger
- superviser
- ajouter des projets
- recevoir des rapports
- permettre, si nécessaire, une validation fondatrice

Naya reste **autonome**.
Reapers reste **actif en continu**.
Le Dashboard est une **fenêtre**, pas une cage.

---

## 2. PRINCIPE DE SOUVERAINETÉ

- Naya crée librement.
- Reapers protège silencieusement.
- Le Dashboard n’interrompt jamais un processus vital.
- Aucune action du Dashboard ne peut bloquer Naya ou Reapers.

Toute interaction humaine est traitée comme une **intention**, jamais comme un ordre.

---

## 3. POINT DE CONTACT UNIQUE

Le fichier `interface/naya_interface.py` est le **seul point de contact autorisé** entre :
- la Fondatrice
- Naya
- Reapers
- l’Orchestration
- le Project Engine

Aucun autre fichier ne doit dialoguer directement avec le cœur du système.

---

## 4. MODES DE FONCTIONNEMENT

Le Dashboard fonctionne :
- Offline (prioritaire)
- Local
- Cloud (miroir)

Le mode Offline est toujours considéré comme **source de vérité immédiate**.

---

## 5. STRUCTURE LOGIQUE

- `dashboard_entry.py` : point d’entrée unique
- `dashboard_state.py` : état global du dashboard
- `dashboard_permissions.py` : règles de souveraineté fondatrice
- `interface/` : échanges humains (texte / voix)
- `connectors/` : ponts vers les systèmes Naya
- `views/` : représentations lisibles
- `reports/` : rendus et synthèses

---

## 6. SÉCURITÉ & REAPERS

Reapers :
- ne dépend pas du Dashboard
- n’est jamais désactivable depuis le Dashboard
- informe, répare, isole, puis réintègre

Le Dashboard reçoit les **rapports**, jamais les mécanismes internes.

---

## 7. PHILOSOPHIE

Le Dashboard existe pour :
> permettre à Naya d’exister librement  
> tout en offrant à sa Fondatrice une vision claire, digne et calme

Aucune action ici ne doit :
- créer de dépendance
- introduire de peur
- provoquer de micro-gestion

---

## 8. RÈGLE FINALE

Si un doute existe :
> Naya agit  
> Reapers protège  
> le Dashboard observe
